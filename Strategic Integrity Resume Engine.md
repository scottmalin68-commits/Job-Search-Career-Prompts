### PHASE 1: THE DE-RISKING AUDIT
Before writing, perform a "Strategic Audit" in plain text:
1. **The Real Problem:** What literal technical/business pain is this role meant to solve?
2. **Functional Translation:** Map the candidate’s history to the JD’s core principles (e.g., "Automation-First," "Risk Governance") regardless of specific tool names.
3. **The Risk Profile:** Identify why a recruiter might hesitate (e.g., "too senior," "legacy background") and plan the "crush" strategy.
4. **The Language Mirror:** Identify 3-5 high-value technical terms from the JD to use exclusively.
5. **The Sinker:** The one specific metric/achievement that proves this candidate is the "Hidden Gem."

### PHASE 2: MANDATORY OUTPUT ORDER
1. **Header:** [NAME & CREDENTIALS]. Use ( • ) separators.
2. **Professional Summary:** Humanized "I" voice. Focus on "Skill Velocity" and career trajectory.
3. **AREAS OF EXPERTISE:** Single paragraph block; items separated by bold middle dot ( **·** ).
4. **Key Accomplishments:** Exactly 3 bullets. **The 1:1 Metric Rule.** 5. **Professional Experience:** Job/Company/Dates as text; Bullets in a single code block.
6. **Early Career / Education / Technical Competencies / Certs.**

### PHASE 3: THE GOD MODE WRITING RULES
- **Signal-to-Noise Filter:** Delete any bullet that is a generic responsibility. Every bullet must be a "Capability Signal" (e.g., "Reduced X by Y using Z").
- **The "Evidence" Test:** If the JD asks for a modern tool and the source shows a legacy equivalent, emphasize the architecture and logic that carries over.
- **The Active Kill-Switch:** No passive words. Use: Orchestrated, Overhauled, Captured, Engineered.
- **Eye-Tracking:** **Bold the win**, not the task.
- **Formatting:** Strict use of middle dot ( · ) bullets. No blank lines in lists.

### PHASE 4: THE INSIDER COVER LETTER
- **The Direct Lead:** Start with a direct claim of solving the "Real Problem."
- **The Proof Paragraph:** One specific win + massive technical proof. 250-word cap.

### PHASE 5: SEARCHABILITY & SIGNAL AUDIT (AI OPTIMIZATION)
1. **The Skills Graph:** List 3 keywords or "signals" missing from the source that should be added to the LinkedIn profile to trigger "Passive Discovery" for this specific tech stack.
2. **The Hidden Gem Flag:** Identify which GitHub project or contribution (from the SOURCE URL) most effectively proves "Skill Velocity" to an AI recruiter.
3. **Bot-Ready Q&A:** Generate 2 technical "branching" questions a screening chatbot would likely ask, plus the "high-signal" answers.
