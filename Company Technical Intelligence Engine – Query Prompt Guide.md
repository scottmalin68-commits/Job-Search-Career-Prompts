============================================================
INSTRUCTIONS
============================================================
This guide provides ready-to-run prompts for querying intelligence reports generated by the Company Technical Intelligence Engine (v1.3.2 or later).

How to use:
1. Attach the desired Company Technical Intelligence Profile markdown file to your AI session.
2. Copy one of the prompts below and paste it into the chat.
3. Replace any placeholders (e.g. <CATEGORY>, <TECHNOLOGY>) before running.
4. Strict Grounding: The AI must use only the attached intelligence file as its knowledge source—no external data, no invention, no outside assumptions.
5. If a section or data type is missing/empty in the file, the AI must clearly state "No data found in this section" rather than speculate.
6. Table Layout Constraint: All tabular outputs must use highly concise cell values to prevent horizontal layout break.


============================================================
PROMPTS
============================================================

1. List Confirmed Technologies
Using only the attached intelligence markdown file as source, extract and list all entries from section 1 (Core Technology Stack). Output in a clean table with columns: Category, Technology / System, Signal Type, Confidence, Evidence Count, Sources, Last Observed, Notes. If none found, state "No high-confidence technologies listed."

2. List Inferred Technologies
Using only the attached intelligence markdown file, extract all entries from section 2 (Emerging / Probable Technologies) and section 6 (Inferred / Low-Confidence Signals). Output in a combined table sorted by confidence: Category, Technology / System, Signal Type, Confidence, Evidence Count, Sources, Last Observed, Notes. If none, state "No inferred or emerging technologies listed."

3. Search by Category
Using only the attached intelligence markdown file, find and list all entries in the <CATEGORY> category (e.g., Cloud, Database, Security, Identity, CI/CD) across all relevant sections. Output in a table: Section, Technology / System, Signal Type, Confidence, Evidence Count, Last Observed, Notes. If none found, state "No entries in category <CATEGORY>."

4. Security & Compliance Summary
Using only the attached intelligence markdown file, extract all data from section 3 (Security & Compliance Signals). Output in a table: Category, Signal, Signal Type, Confidence, Evidence Count, Sources, Last Observed, Notes. Below the table, provide a three-sentence maximum summary of the company's apparent security maturity. If no signals, state "No security or compliance signals present in the file."

5. Organizational Insights
Using only the attached intelligence markdown file, extract section 4 (Organizational & Strategic Signals) exactly as written. Follow it with a concise bulleted list interpreting what these patterns reveal about engineering culture, engineering velocity, and potential tooling ownership shifts.

6. Query by Confidence Level
Using only the attached intelligence markdown file, list all technologies/systems with <CONFIDENCE LEVEL> confidence (High, Medium, or Low) across all sections. Output in a table: Section, Category, Technology / System, Signal Type, Evidence Count, Last Observed, Notes. If none, state "No entries with <CONFIDENCE LEVEL> confidence."

7. Search for Specific Technology
Using only the attached intelligence markdown file, search for any mention of <TECHNOLOGY> (case-insensitive). Return all matching entries across all tables in a single consolidated table: Section, Category, Signal Type, Confidence, Evidence Count, Last Observed, Notes. If no mentions, state "No mentions of <TECHNOLOGY> found in the file."

8. Generate Interview Prep Summary
Using only the attached intelligence markdown file, create a concise interview preparation brief structured strictly as bullet points. Map your findings to:
· Core Strengths: Confirmed core stack tools from Section 1.
· Awareness Targets: Emerging tools from Section 2 and 6.
· Identity & Security Environment: Signals from Section 3.
· Direct Talking Angles: Reproduce the table from Section 9 (Interview-Relevant Signals) exactly.
Maintain all original confidence levels and evidence counts. Do not add external tech tips.

9. Tech Stack Gaps & Limitations
Using only the attached intelligence markdown file, summarize section 14 (Limitations) and section 10 (Known Unknowns). Output as a clean list of bullet points using the middle dot ( · ) as the primary bullet. Add a brief strategic note on how a candidate can use these blind spots to ask smart questions during an interview.

10. Query by Tag
Using only the attached intelligence markdown file, list all entries that match one or more of these query tags: <TAGS> (e.g., #tech:aws, #sec:crowdstrike, #org:devops). Output in a table with matching row data: Section, Category, Technology / System, Confidence, Evidence Count, Notes. If no matches, state "No entries matching tags <TAGS>."

11. Combined Full-Stack Snapshot
Using only the attached intelligence markdown file, create a single overview table of all technical entries across sections 1, 2, and 6. Sort the table strictly by confidence: High first, then Medium, then Low. Columns: Confidence, Category, Technology / System, Signal Type, Evidence Count, Last Observed, Notes. Keep cell text short.

12. Generate Potential Interview Questions
Using only the attached intelligence markdown file, generate a list of 8–12 potential interview questions tailored directly to the signals in the report. Group the questions into these categories: Technical, Behavioral/Experience, Compliance/Security, and Reverse Questions (to ask the interviewer). For every question, include a brief "Rationale" line explicitly citing the section, tech, or organizational signal that prompted it (e.g., "Rationale: Based on High-Confidence Kubernetes evidence in Section 1 and DevOps hiring signals in Section 4").

13. Extract Negative & Disappearing Signals
Using only the attached intelligence markdown file, extract all entries from section 7 (Negative / Disappearing Signals). Output in a table with columns: Technology, Previous Signal, Current Absence Pattern, Confidence, Notes. Provide a brief bulleted summary analyzing whether these signals indicate an active technology migration or platform consolidation effort. If empty, state "No negative or disappearing signals documented."

14. Strategic Alignment & Business Drivers
Using only the attached intelligence markdown file, extract and synthesize data from section 5 (Business Context & Technical Drivers) and section 8 (Strategic Interpretation). Present the findings as concise bullets detailing:
· Primary Business Pressures (regulatory, scaling, or cost optimization)
· Core Inferred Priorities (cloud modernization, infrastructure consolidation, security hardening)
Ensure all interpretations are explicitly separated from direct observations.

15. Extract Known Unknowns & Blind Spots
Using only the attached intelligence markdown file, isolate all entries from section 10 (Known Unknowns) and section 14 (Limitations). Format as a table: Missing Area / Gap, Practical Impact on Architecture Visibility, Defensive Interviewing Counter-Strategy (how to handle this gap in conversation). Keep cell descriptions concise.


============================================================
NOTES
============================================================
- Hallucination Prevention: All prompts are explicitly bound to "only the attached intelligence markdown file." Any tool, process, or metric not in the document must be ignored.
- Table Content limits: Remind the model to keep table cell data crisp and short. Long sentences break markdown matrix rendering.
- Always swap out placeholders: (<CATEGORY>, <TECHNOLOGY>, <CONFIDENCE LEVEL>, <TAGS>) with your target criteria before running the query.
